# 🚀 إعداد قاعدة بيانات Supabase للطلبات والفواتير

لقد قمنا بربط الموقع بـ **Supabase** لتخزين الطلبات والفواتير سحابياً وبشكل دائم ومجاني 100%!

إليك الخطوات البسيطة لتفعيل الربط وبدء استخدام السحابة:

---

## 1️⃣ إنشاء حساب وقاعدة بيانات مجانية على Supabase
1. اذهب إلى موقع [Supabase.com](https://supabase.com) وسجل دخولك (مثلاً باستخدام حساب GitHub مجاني).
2. قم بإنشاء مشروع جديد **(New Project)** واختر اسماً له وحدد كلمة مرور قوية لقاعدة البيانات والمنطقة الأقرب إليك.
3. انتظر دقيقة واحدة حتى تكتمل عملية إنشاء المشروع.

---

## 2️⃣ تهيئة الجدول داخل قاعدة البيانات (SQL Editor)
1. في القائمة الجانبية للـ Supabase، اضغط على **SQL Editor** (أيقونة المربع الذي يحتوي على رمز `>_`).
2. اضغط على **New query** (استعلام جديد).
3. افتح ملف `supabase_setup.sql` الموجود في هذا المجلد، وانسخ كل محتواه.
4. ألصق محتوى الكود البرمجي في محرر الاستعلامات في Supabase واضغط على **Run** في أسفل اليمين.
5. ستظهر لك رسالة تأكيد النجاح: `Success. No rows returned.`. ممتاز! لقد تم إنشاء جدول `orders` المخصص للطلبات بنجاح.

---

## 3️⃣ ربط مفاتيح الاتصال في الإعدادات (Environment Variables)
للبدء الفوري، اذهب إلى إعدادات السحابة في منصة العمل **(Settings / Secrets)** في AI Studio، أو أضفها في ملف البيئة الخاص بك:

أضف المتغيرات التالية:
* **`SUPABASE_URL`**: ستجده في Supabase تحت (Project Settings ⚙️ -> API -> Project URL).
* **`SUPABASE_SERVICE_ROLE_KEY`**: ستجده تحت (Project Settings ⚙️ -> API -> `service_role` key - `secret`). هذا المفتاح يُستخدم بشكل آمن وحصري في الخلفية (Backend) دون كشفه للمتصفح.

---

## 🧠 ميزة الأمان العالي والدعم الثنائي (Dual Sync & Fallback)
* **أمان متكامل**: الربط يتم بالكامل في كود الخلفية (Backend - Server-Side) مما يحمي مفاتيح Supabase من السرقة أو العبث بواسطة المتصفحات.
* **استقرار دائم**: الكود يضم آلية احتياطية ذكية (Fallback)؛ في حال عدم إضافة مفاتيح Supabase بعد، سيستمر الموقع في حفظ الطلبات محلياً في ملف `data/orders.json` دون أي توقف، وفور إضافتك للمفاتيح سينتقل تلقائياً للتخزين السحابي الآمن!
